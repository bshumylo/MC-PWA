# Ручна перевірка на живому сервері

## Передумови

- [ ] HTTPS з валідним сертифікатом
- [ ] Працює cron або daemon (Адміністрація → Задачі — черга q0 обробляється)
- [ ] PHP-розширення: openssl, curl, gd (стандартні вимоги EspoCRM)

## 1. Встановлення

- [ ] Адміністрація → Розширення → завантажити `mc-pwa-1.0.0.zip` → Встановити
- [ ] `php tests/webpush-selftest.php` на сервері → `ALL TESTS PASSED`
- [ ] Адміністрація: з'явився розділ **PWA** (Налаштування, Встановлення, Push-підписки)
- [ ] У Налаштуваннях PWA заповнено «Публічний ключ VAPID» (генерується автоматично)

## 2. PWA-встановлюваність

- [ ] Відкрити сайт у Chrome (Android) → меню → «Додати на головний екран» / банер встановлення
- [ ] `https://your-espocrm.example.com/?entryPoint=pwaManifest` → коректний JSON-маніфест
- [ ] `https://your-espocrm.example.com/?entryPoint=pwaIcon&size=192` → PNG-іконка
- [ ] DevTools → Application → Manifest: без помилок, іконки видно
- [ ] DevTools → Application → Service Workers: зареєстрований, scope `/`

## 3. Швидкі дії (App Shortcuts)

- [ ] Встановити додаток на Android → довге натискання на іконку → пункти з Quick Create List
- [ ] Тап по пункту (напр. «Контакт») → відкривається форма створення

## 4. Push-сповіщення

- [ ] Відкрити встановлений додаток, увійти, торкнутись екрана → запит дозволу → Дозволити
- [ ] Адміністрація → PWA → Push-підписки: з'явився запис для користувача
- [ ] З іншої обліковки призначити користувачу задачу (Assigned User)
- [ ] Закрити додаток → сповіщення приходить на пристрій (протягом хвилини, залежить від cron)
- [ ] Тап по сповіщенню → відкривається відповідний запис
- [ ] Згадка @username у стрічці → приходить push «Вас згадали»
- [ ] Вимкнути тип у «Типи push-сповіщень» → сповіщення цього типу не надходять

## 5. Статистика

- [ ] Увімкнути «Статистика встановлень» у Налаштуваннях PWA
- [ ] Перевстановити/відкрити додаток на телефоні
- [ ] Адміністрація → PWA → Встановлення PWA: запис із платформою, типом пристрою, версією ОС
- [ ] У записі немає жодних даних користувача (лише анонімний ID)

## 6. Безпека

- [ ] `?entryPoint=pwaSubscribe` GET-запитом → помилка (POST expected)
- [ ] POST на `pwaSubscribe` без заголовка `X-Requested-With` → 403
- [ ] POST на `pwaSubscribe` без логіну (інкогніто) → 401/помилка авторизації
- [ ] Під звичайним користувачем: `#PwaSubscription` та `#PwaInstallation` недоступні (ACL)
- [ ] REST API: `GET /api/v1/Settings` під звичайним користувачем не містить `pwaVapidPrivateKey`

## 7. iOS (за наявності пристрою, iOS 16.4+)

- [ ] Safari → Поділитись → «На початковий екран»
- [ ] Відкрити додаток → увійти → торкнутись екрана → дозволити сповіщення
- [ ] Push приходить при закритому додатку
